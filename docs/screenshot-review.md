# Screenshot Review Checklist

Use this checklist before publishing any UI screenshots.

## Review for Sensitive Details

- private IP addresses
- MAC addresses
- internal hostnames
- usernames
- storage identifiers that should stay private
- browser tabs or sidebars with private context
- secrets visible in forms or query strings

## Review for Operational Noise

- temporary test names
- stale labels
- unrelated services
- personal bookmarks or profile details

## Review for Clarity

- crop to the relevant area when possible
- prefer screenshots that explain structure rather than expose clutter
- use captions in docs so readers know why the screenshot matters

## Publication Decision

- if a screenshot needs heavy redaction, it may be better to replace it with a diagram
- if a screenshot is not clearly useful, leave it out
